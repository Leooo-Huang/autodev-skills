# 步骤 2：增量创意分析

**目标**：只对新功能做创意分析，不重复已有功能的 ideation。

## 执行

1. **明确新功能边界**：
   - 新功能解决什么问题？目标用户是谁？
   - 与现有功能的关系：独立？扩展？还是依赖某个已有功能？

2. **轻量调研**（不需要像 `/autodev-ideation` 那样做完整三维调研）：
   - 竞品有类似功能吗？怎么做的？
   - 有没有已知的最佳实践？
   - 如果新功能涉及新技术，WebSearch 验证可行性

3. **确定新功能的 P/W 编号**：
   - 读取现有 ideation.md 的最大编号
   - 新功能是平台层能力（P）还是业务场景（W）？
   - 分配新编号（如现有到 P3/W5，新功能是 P4 或 W6）

4. **检查与现有功能的 MECE 关系**：
   - 新功能和已有功能有重叠吗？
   - 新功能依赖哪些已有功能？
   - 参考 `autodev-ideation/checklists/mece-check.md`

## 必须输出

向用户展示：
- "新功能 {名称}，编号 {P/W编号}"
- "与现有功能的关系：{依赖/独立/扩展}"
- "核心用户故事：{1-3 条}"

## 追加到文档

在 `*-ideation.md` 的 MVP 表格末尾追加新功能条目（不修改已有条目）。
