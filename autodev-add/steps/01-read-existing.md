# 步骤 1：读取现有设计文档

**目标**：理解当前项目的完整架构和功能版图，为增量设计提供上下文。

## 执行

1. **读取精炼摘要**（优先）：
   - `*-index.md` — 项目全景、关键决策、能力-组件映射
   - `*-rules.md` — 编码约束、API 约定

2. **读取功能清单**：
   - `*-ideation.md` — 现有 MVP 功能列表（P/W 编号）、场景依赖图
   - 记录：当前有哪些功能、哪些已完成、哪些在进行中

3. **读取架构**：
   - `*-design.md` — 技术栈、架构模式、组件划分、能力-组件映射表
   - 记录：新功能应该放在哪个组件/模块下，还是需要新建组件

4. **快速扫描代码现状**：
   - `git log --oneline -20` — 最近的开发方向
   - 目录结构 — 现有的文件组织方式

## 必须输出

向用户展示：
- "当前项目有 N 个功能：{列表}"
- "架构是 {技术栈}，{架构模式}"
- "你要添加的新功能是：{用户输入}"
- "初步判断：新功能应该 {融入现有模块 X / 新建独立模块}"
