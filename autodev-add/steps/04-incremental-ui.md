# 步骤 4：增量 UI 设计

**目标**：设计新功能的 UI/UX，与现有页面风格和导航保持一致。

## 执行

1. **读取现有 UI 设计**：
   - `*-ui.md` — 现有页面清单、信息架构、导航模式
   - 理解现有的设计语言和交互模式

2. **确定新功能的 UI 形态**：
   - **新页面**？→ 放在导航的什么位置？
   - **扩展现有页面**？→ 加在哪个区域？会不会信息过载？
   - **弹窗/面板**？→ 从哪个入口触发？

3. **设计新功能的页面**：
   - 按现有 ui.md 的格式，为新页面/组件撰写：
     - 布局区域划分
     - 组件清单
     - 状态设计（加载/空/正常/错误）
     - 用户流程

4. **导航一致性检查**：
   - 新页面在信息架构中的位置符合用户心智模型吗？
   - 参考 `autodev-ui/checklists/ui-completeness.md`

## 追加到文档

在 `*-ui.md` 中追加新章节：
- 信息架构更新（如有新的导航项）
- 新页面/组件的完整设计
- 新增的用户流程
