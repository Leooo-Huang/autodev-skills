# 阶段 4：设计方案（plan mode）

**所有修改都必须先展示方案再实现。** 根据复杂度选择方案详细程度：

- **简单修复**（1-2 文件，改动明确）→ 简要方案（向用户展示：改哪个文件、改什么、为什么），确认后实现
- **涉及 UI 交互**（任何用户可见的改动）→ **必须** plan mode，方案须包含交互设计
- **复杂修复**（涉及架构变更）→ plan mode + 用户讨论

**方案必须包含**：
1. 要改的文件和具体改动
2. 为什么这样改（根因 → 方案的逻辑链）
3. **交互设计**（如涉及 UI）：具体的手势、状态变化、视觉反馈
4. **功能迁移形态**（如涉及"把功能从 A 移到 B"）：
   - 目标页面当前有多少信息？加入后会不会过载？
   - 评估三种形态：**内联**（直接放入现有列表） vs **子页面**（点击跳转） vs **展开面板**（点击展开详情）
   - 决策标准：<=3 项设置 → 内联；4-8 项 → 展开面板；>8 项或需要预览效果 → 子页面
   - 新增内容的视觉风格必须与目标页面现有元素一致
5. 验证方式
