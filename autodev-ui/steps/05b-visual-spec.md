# 步骤 5b：视觉规范（L3）

**目标**：定义项目的视觉设计语言，确保实现阶段不用"猜"配色、字体、间距。

## 决策来源

1. 读取 `*-design.md` 中的技术选型——确认使用的 UI 框架（shadcn/MUI/Ant/自定义）
2. 如果用户提供了品牌指南或偏好，以用户输入为准
3. 如果没有，基于产品调性自动推荐：
   - 科技/效率类产品 → 冷色（蓝/紫）+ 直角/小圆角 + 紧凑密度
   - 消费/生活类产品 → 暖色（橙/绿）+ 大圆角 + 舒适密度
   - 企业/B2B 产品 → 中性（蓝灰）+ 小圆角 + 紧凑密度

## 5 个核心决策

### 1. 品牌色

定义色彩系统，产出格式：

| 角色 | 色值 | CSS 变量 | 用途 |
|------|------|---------|------|
| 主色 | {hex} | `--primary` | 按钮、链接、焦点环、选中态 |
| 主色悬停 | {hex} | `--primary-hover` | 按钮悬停态 |
| 辅助色 | {hex} | `--accent` | 标签、徽章、强调元素 |
| 成功 | {hex} | `--success` | 成功状态、完成标记、正向操作 |
| 警告 | {hex} | `--warning` | 警告提示、需注意的状态 |
| 错误 | {hex} | `--destructive` | 错误状态、删除操作、危险操作 |
| 背景 | {hex} | `--background` | 页面背景 |
| 卡片背景 | {hex} | `--card` | 卡片、面板背景 |
| 边框 | {hex} | `--border` | 分割线、输入框边框 |
| 文字-主要 | {hex} | `--foreground` | 标题、正文 |
| 文字-次要 | {hex} | `--muted-foreground` | 辅助文字、占位符 |

如果 UI 框架是 shadcn，直接给出 HSL 值以匹配其 CSS 变量命名。

### 4. 间距与信息密度

基础单位：{4px}
间距刻度：`{4 / 8 / 12 / 16 / 24 / 32 / 48 / 64}px`

| 场景 | 间距 | 说明 |
|------|------|------|
| 组件内边距 | {值} | 按钮、卡片内部 |
| 组件间距 | {值} | 同级元素之间 |
| 区域间距 | {值} | 页面大区块之间 |
| 页面边距 | {值} | 内容区与屏幕边缘 |

信息密度：{紧凑 / 舒适 / 宽松}

### 5. 阴影与层级

| 层级 | 阴影值 | 用途 |
|------|--------|------|
| 无 | none | 扁平元素 |
| 低 | `0 1px 2px rgba(0,0,0,0.05)` | 卡片默认 |
| 中 | `0 4px 6px rgba(0,0,0,0.07)` | 悬浮卡片、下拉菜单 |
| 高 | `0 10px 15px rgba(0,0,0,0.1)` | 模态框、弹窗 |

## 额外决策（可选）

- 暗色模式：{支持 / 不支持 / 后续考虑}
- 图标风格：{线性 / 填充}，图标库：{Lucide / Heroicons / 自定义}
- 插图风格：{无 / 简约线条 / 3D / 手绘}

## 规则

- 如果 UI 框架有默认主题（如 shadcn 的 default/new-york），优先用默认值，只覆盖品牌色
- 色值必须同时提供亮色和暗色版本（如果支持暗色模式）
- 所有数值用 CSS 变量表达，不硬编码到组件中
